## [5.0.0] - 2026-05-03

### Додано (PWA та Web Push)
- **Web App Manifest** з повною конфігурацією PWA
- **Service Worker** з кешуванням статичних ресурсів
- Версійне кешування (versioned cache) з автоматичним оновленням
- **Push сповіщення** через Web Push API
- VAPID ключі для захищених push-повідомлень
- Збереження push-підписок у JSON файлі (persistent storage)
- Сповіщення про підключення нового користувача
- Інтерфейс для запиту дозволу на сповіщення
- Відображення статусу PWA компонентів в інтерфейсі

### Змінено
- Оновлено клієнтський код для підтримки Service Worker
- Додано API ендпоінти для управління push-підписками
- Розширено статистику в інтерфейсі

## [6.0.0] - 2026-05-25 16:52

### Додано (Celery фонові задачі)
- **Celery інтеграція** з Redis як broker та result backend
- **Фонова задача експорту** (export_grid_task) для генерації JSON-експорту стану поля
- **Idempotent поведінка** - повторний експорт однакового стану повертає кешований результат
- **HTTP-endpoint** для запуску експорту (POST /api/export/start) з поверненням 202 Accepted
- **HTTP-endpoint** для перевірки статусу задачі (GET /api/export/status/<task_id>)
- **HTTP-endpoint** для завантаження готового файлу (GET /api/export/download/<task_id>)
- **HTTP-endpoint** для списку експортів (GET /api/export/list)
- **Стійке збереження метаданих** задач у JSON файл (	asks_metadata.json)
- **Періодична задача** очищення застарілих експортів (щодня о 2:00, зберігає 7 днів)
- **Періодична задача** перерахунку агрегованої статистики (кожні 6 годин)
- **UI панель експорту** зі стартом задачі, ручним опитуванням та індикацією станів
- **Візуальна індикація прогресу** (PENDING, STARTED, SUCCESS, FAILURE, RETRY)
